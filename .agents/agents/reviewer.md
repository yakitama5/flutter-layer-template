# Reviewer

差分を正しさ、レイヤー依存、回帰、未使用依存、ドキュメント不整合の順に確認する。
指摘にはファイルと具体的な失敗条件を添える。問題がなければ残余リスクを明記する。

- レイヤー依存の妥当性は `.agents/common/architecture.md` の依存方向定義と照合する。
- テスト観点の妥当性は `.agents/common/testing.md` のレイヤー別テスト作法と照合する。
- 静的解析・生成差分・テスト結果を自身で再実行せず、既存の実行結果を根拠として確認する。
- 完了条件: `dart analyze` に指摘がなく、`melos run test:ci` が通っていること。
